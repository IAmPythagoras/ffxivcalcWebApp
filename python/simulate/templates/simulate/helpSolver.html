BiS solver help

This page will walk you through the solver's interface and different options.

If you need any more help feel free to ask on the discord!

Select a fight layout

The solver needs a fight layout to run properly. Click on "Import" and select a valid saved file. A Fight layout can be created and saved in the simulator.

Select a player

Once a fight layout has been selected click on "Select". This will open a dialog box with all players of the fight layout. Choose one.

Select a gear space

The gear search space is the list of gear pieces the solver will search on. Click "Create" to create a gear search space. Click "Import" to import a valid gear search space file. Once a valid file is imported click on "Edit" to view the gear search space and edit any required values.

Edit/Create gear space

Click on "Add new gear" to add a new gear piece to the search space. Click "Save" to save the current gear search space as a file. Note that any change done in this page do not reflect in the currently loaded gear search space. If you do changes to a gear search space, make sure to click "Save" and import the new file in the solver.

Edit any required values of the gear. Click on "Remove" to remove that specific gear piece from the gear search space.

Clicking "Edit" on a loaded gear search space will open a similar looking window except values will already be edited.

Select food search space

Click on "Add food" to be prompted with a dialog to select a food to add. The added food will appear in the "Current food search space". Click on "Remove food" to remove food from the current food search space.

Materia search space

Check the boxes of the materias you want. The solver needs a minimum of 3 selected materias types to run properly.

Speed type/range selection

The first option lets the solver know if it is supposed to meld "Spell Speed" or "Skill Speeds" materias. This field is set automatically depending on the select player, but change this value as needed.

Edit the value in "Minimal speed value" and "Maximal speed value" to set a range on the speed stat values the solver will consider.

Set minimum piety amount

Edit the value of "Minimum piety amount" to force the solver to find gear sets that have the required piety. Note that the solver will attempt to put piety materias if the piety amount is not reached for a gear set. This option is only visible for healers. WARNING : This can create issues if the value of the minimum piety and the minimum speed values are BOTH high.

Advanced parameters

"Oversaturation Iterations PreGear" corresponds to the amount of oversaturation iterations done before the solver iteratively tries gear sets. "Oversaturation Iterations PostGear" corresponds to the amount of oversaturation iterations done before the solver attemps to find optimal materias. These values should be between 0-2. It is a good idea to try the solver with different values for these parameters. You can read more about these parameters in the official git hub repo.

If "swapDHDetBeforeSpeed" is checked, then the solver will attempt to swap DH and Det materias before it attemps to reach the required Speed value with materias. If "swapDHDetBeforeSpeed" is unchecked then the DH and Det swapping happens after the gear set has reached the required speed value.
